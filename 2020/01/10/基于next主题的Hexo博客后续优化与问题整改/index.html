换成 next 主题

  • 在源目录 \Blog\blog 右键选择 Git Bash Here

  • 输入 git clone https://github.com/theme-next/hexo-theme-next themes/next

  • 完成后打开站点配置文件 \Blog\blog_config.yml

  • 把 theme 修改为 next ，如下图

  • next主题目前有四个风格，我是采用 Gemini，可以都试试，在本地三部曲预览

    • hexo clean
    • hexo g
    • hexo s

  • 选择的话只要去掉注释 ‘#’，再把不要的加上注释 ‘#’ 就OK了

关于侧栏标签，分类归档等打开后显示 Cannot GET /xxxx 问题的解决

这问题一度把我整晕了，可算查找解决办法到崩溃

  • 首先确保创建步骤
    1. 在 \Blog\blog 下操作命令行，输入：
      • hexo new page "tags"
      • hexo new page "categories"
      • hexo new page "archives"
    2. 进入 \Blog\blog\source
      • 可以看到生成了许多新文件夹
      • 进入刚刚生成的子文件夹，比如 \tags
      • 添加代码
      • 其他几项同理，修改关键字即可
    3. 打开主题配置文件 \Blog\blog\themes\next_config.yml
      • 找到 menu:
      • 把你想要的功能前的‘＃’注释去掉，可以效仿下图
    4. 还没结束，非常重要的一点：拿 tags 举例
      • tags: /tags/|| tags
      • /tags/ 要与其后的’||’紧挨着，之间绝对不能有空格，这是产生 Cannot GET /xxxx 问题的究极大源头
    5. 不出意外应该可以了，可以三部曲在本地试试
      • hexo clean
      • hexo g
      • hexo s
      • 在浏览器输入 localhost:4000 试试如果没问题，再部署到远端
      • hexo d
